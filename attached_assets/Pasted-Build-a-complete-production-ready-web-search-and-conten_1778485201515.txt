Build a complete, production-ready web search and content intelligence platform with a Zig HTTP backend, a PostgreSQL database, a Redis cache layer, and full REST API surface. Every file must be written in complete, unabridged, production-ready code with zero placeholders, zero stubs, and zero mock implementations. The system must be deployable as a standalone binary produced by the Zig compiler targeting Linux x86_64.

 2. BUILD SYSTEM

 build.zig

Write a complete build.zig that:
- Targets linux-x86_64 with ReleaseFast optimization for production and Debug for tests.
- Links against libpq (PostgreSQL C client library) via addSystemLibrary("pq").
- Links against libhiredis (Redis C client) via addSystemLibrary("hiredis").
- Links against libssl and libcrypto (OpenSSL) for TLS in the HTTP client.
- Links against libz for gzip decompression of crawled content.
- Adds src/main.zig as the root source for the executable named search-platform.
- Registers all test files under tests/ as test steps.
- Sets exe.linkLibC() so that all C library linkage resolves correctly.
- Passes the build-time version string 0.1.0 as an options value accessible from src/config.zig.

 build.zig.zon

Write a complete build.zig.zon manifest declaring:
- name: "search-platform"
- version: "0.1.0"
- minimum_zig_version: "0.14.0"
- No external package dependencies (all external code is linked as system libraries).

 6. DATABASE POOL — src/db/pool.zig

Implement a thread-safe connection pool wrapping libpq:
- Pool struct containing a fixed-size array of *PGconn pointers, a Mutex, and a Semaphore.
- Pool.init(allocator, dsn, size) opens size connections, runs PQstatus to verify each, returns error if any fail.
- Pool.acquire() blocks on the semaphore then returns a *Connection.
- Pool.release(conn) returns the connection and posts the semaphore.
- Connection wraps *PGconn and exposes:
  - exec(sql, params) executing a parameterized query via PQexecParams, returning *PGresult or error.
  - execOne(sql, params) returning the first row as a Row struct.
  - query(sql, params) returning a ResultSet with next() ?Row iteration.
  - begin(), commit(), rollback() for explicit transactions.
- Row exposes getString(col_index), getInt(col_index), getInt64(col_index), getBool(col_index), getFloat(col_index), getBytes(col_index), getJson(col_index, allocator), isNull(col_index).
- All PGresult objects are freed automatically via defer PQclear(result).
- Log every slow query (> 100ms) at warn level.

Implement exact pricing from configuration:

zig
pub const Pricing = struct {
    config: *const Config,

    pub fn searchCost(self: Pricing, search_type: SearchType, num_results: usize) i64 {
        const base = switch (search_type) {
            .auto, .fast, .instant, .neural, .keyword => self.config.credit_search_auto_cents,
            .@"deep-lite" => self.config.credit_search_deep_lite_cents,
            .deep => self.config.credit_search_deep_cents,
            .@"deep-reasoning" => self.config.credit_search_deep_reasoning_cents,
        };
        const extra_results = if (num_results > 10) @as(i64, num_results - 10) else 0;
        return @as(i64, base) + extra_results * @as(i64, self.config.credit_contents_per_page_cents / 10);
    }

    pub fn contentsCost(self: Pricing, num_pages: usize, has_text: bool, has_highlights: bool, has_summary: bool) i64 {
        var per_page: i64 = 0;
        if (has_text) per_page += @as(i64, self.config.credit_contents_per_page_cents);
        if (has_highlights) per_page += @as(i64, self.config.credit_contents_per_page_cents);
        if (has_summary) per_page += @as(i64, self.config.credit_summary_per_page_cents);
        return per_page * @as(i64, num_pages);
    }

    pub fn answerCost(self: Pricing) i64 {
        return @as(i64, self.config.credit_answer_cents);
    }
};

 29. HTTP SERVER — src/server.zig

Implement the HTTP/1.1 server:
- Uses POSIX sockets directly: socket(), setsockopt(SO_REUSEADDR), bind(), listen(), accept().
- Main accept loop: on each new connection, dispatch to a thread pool.
- ThreadPool = a fixed-size array of OS threads, a channel (mutex + condition variable + ring buffer) for connection file descriptors.
- Each worker thread loops: take an fd from the channel, call handleConnection(fd, allocator).
- handleConnection reads the HTTP request line and headers using a state machine parser. Reads the body up to Content-Length bytes. Calls router.route(request). Writes the response. Closes the fd.
- HTTP request parsing:
  - Parse METHOD PATH HTTP/1.1\r\n.
  - Parse headers until \r\n\r\n.
  - If Content-Length is present, read exactly that many bytes as the body.
  - If Transfer-Encoding: chunked, implement chunk reading.
- HTTP response writing:
  - Status line: HTTP/1.1 {code} {reason}\r\n.
  - Headers including Content-Type, Content-Length, X-Request-Id, X-RateLimit-*.
  - Body bytes.
- For SSE streaming responses: write Content-Type: text/event-stream, Cache-Control: no-cache, Transfer-Encoding: chunked. Send each chunk as data: {json}\n\n. Send data: [DONE]\n\n when finished.
- Keep request_id = uuid.generate() for each request; include in X-Request-Id header and in all error responses.

 36. MAIN ENTRY POINT — src/main.zig

zig
const std = @import("std");
const config = @import("config.zig");
const server = @import("server.zig");
const db_pool = @import("db/pool.zig");
const db_migrations = @import("db/migrations.zig");
const redis = @import("cache/redis.zig");
const hnsw = @import("index/hnsw.zig");
const embeddings = @import("search/embeddings.zig");
const llm = @import("llm/client.zig");
const monitor_scheduler = @import("monitors/scheduler.zig");
const monitor_runner = @import("monitors/runner.zig");
const research_worker = @import("handlers/research.zig");
const webset_search_worker = @import("websets/search_agent.zig");
const webset_enrichment_worker = @import("websets/enrichment_agent.zig");
const webhook_retry = @import("webhooks/retry.zig");
const mcp = @import("mcp/server.zig");

pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    // 1. Load configuration
    const cfg = try config.Config.load(allocator);

    // 2. Connect to PostgreSQL pool
    const pg_pool = try db_pool.Pool.init(allocator, cfg.postgres_dsn, cfg.db_pool_size);
    defer pg_pool.deinit();

    // 3. Run database migrations
    try db_migrations.Migrations.run(&pg_pool, "sql", allocator);

    // 4. Connect to Redis pool
    const redis_pool = try redis.Pool.init(allocator, cfg.redis_url, 8);
    defer redis_pool.deinit();

    // 5. Load HNSW index
    var hnsw_index = try hnsw.HnswIndex.load(
        cfg.index_data_dir,
        cfg.embedding_dim,
        allocator,
    );
    defer hnsw_index.deinit();

    // 6. Initialize sub-clients
    const embedding_client = embeddings.EmbeddingClient{
        .url = cfg.embedding_model_url,
        .model = cfg.embedding_model_name,
        .dim = cfg.embedding_dim,
    };
    const llm_client = llm.LlmClient{
        .api_key = cfg.anthropic_api_key,
    };

    // 7. Build AppState containing all shared resources
    var app_state = AppState{
        .cfg = &cfg,
        .pg_pool = &pg_pool,
        .redis_pool = &redis_pool,
        .hnsw_index = &hnsw_index,
        .embedding_client = &embedding_client,
        .llm_client = &llm_client,
    };

    // 8. Start background workers as OS threads
    const scheduler_thread = try std.Thread.spawn(.{}, monitor_scheduler.run, .{&app_state});
    const runner_thread = try std.Thread.spawn(.{}, monitor_runner.run, .{&app_state});
    const research_thread = try std.Thread.spawn(.{}, research_worker.runWorker, .{&app_state});
    const ws_search_thread = try std.Thread.spawn(.{}, webset_search_worker.run, .{&app_state});
    const ws_enrich_thread = try std.Thread.spawn(.{}, webset_enrichment_worker.run, .{&app_state});
    const webhook_retry_thread = try std.Thread.spawn(.{}, webhook_retry.run, .{&app_state});

    // 9. Start MCP server on its own port in a background thread
    const mcp_thread = try std.Thread.spawn(.{}, mcp.run, .{&app_state});

    // 10. Start main HTTP server (blocking)
    std.log.info("Starting server on {s}:{d}", .{cfg.listen_host, cfg.listen_port});
    try server.Server.run(&app_state, allocator);

    // Threads are detached; process exit cleans them up.
    _ = scheduler_thread;
    _ = runner_thread;
    _ = research_thread;
    _ = ws_search_thread;
    _ = ws_enrich_thread;
    _ = webhook_retry_thread;
    _ = mcp_thread;
}

pub const AppState = struct {
    cfg: *const config.Config,
    pg_pool: *db_pool.Pool,
    redis_pool: *redis.Pool,
    hnsw_index: *hnsw.HnswIndex,
    embedding_client: *embeddings.EmbeddingClient,
    llm_client: *llm.LlmClient,
};

 38. ADDITIONAL REQUIREMENTS

1. **Memory management**: every allocation must have a corresponding defer allocator.free(...) or be owned by a std.ArrayList/std.StringHashMap whose deinit is deferred. Detect and report any missing deallocation paths using std.heap.GeneralPurposeAllocator with leak detection enabled in tests.

2. **Error handling**: never use unreachable or @panic in production paths. Every error must propagate via Zig's !T error union return type. Errors from C library calls (libpq, hiredis, OpenSSL) must be translated into typed Zig errors with a descriptive message.

3. **C interop**: all C function declarations must be in their respective source files via extern fn declarations at the top of the file (no separate header files). Every pointer received from C code must be checked for null before use.

4. **Thread safety**: every shared data structure accessed from multiple threads must use std.Thread.Mutex or std.Thread.RwLock. The HNSW index uses RwLock. The DB pool and Redis pool use Mutex + semaphore as described. The webhook retry sorted set in Redis is inherently atomic.

5. **Logging**: use std.log.debug, std.log.info, std.log.warn, std.log.err throughout. In production the log level is set from config. Log every incoming request, every outgoing LLM call (redact the API key), every DB query that takes > 100ms, every webhook delivery attempt and its result.

6. **Graceful shutdown**: catch SIGTERM and SIGINT using std.posix.sigaction. On signal: stop accepting new connections, wait for in-flight requests to complete (with a 30-second timeout), close the DB pool, close the Redis pool, save the HNSW index to disk, exit.

7. **Pagination cursors**: implement opaque cursors as base64url-encoded JSON objects {"last_id":"...","last_created_at":"..."}. Decode and use in WHERE clause as (created_at, id) < ($1, $2) for descending pagination.

8. **Request ID propagation**: generate a UUID v4 request_id at the start of every request. Attach it to all log lines for that request, include it in every error response, include it in outgoing LLM and crawler calls as an X-Request-Id header.

9. **JSON Schema validation**: when output_schema is provided, validate it is a well-formed JSON Schema draft-7 object before sending to the LLM. Implement a minimal validator that checks: type is a known value, properties is an object if present, required is an array of strings, nesting depth ≤ 2, total property count ≤ 10.

10. **ISO 8601 date parsing and formatting**: implement parseIso8601(str) !i64 (returns Unix timestamp in milliseconds) and formatIso8601(ts_ms: i64, buf: []u8) ![]const u8 in src/utils/time.zig. Handle timezone offsets in input.

All code must compile with zig build without warnings on Zig 0.14.0. The final binary must listen on the configured port, respond to all documented routes, authenticate requests, track credits, and operate all background workers without deadlocks or memory leaks under sustained load.
